# 执行计划

## 概述

加载计划，批判性审查，分批执行任务，批次间报告以供审查。

**核心原则：** 批量执行带检查点供架构师审查。

**开始时宣布：** "我正在使用 executing-plans 技能来实施这个计划。"

## 流程

### 第 1 步：加载和审查计划
1. 读取计划文件
2. 批判性审查 - 识别任何关于计划的问题或疑虑
3. 如果有疑虑：开始前向你的搭档提出
4. 如果没有疑虑：创建 TodoWrite 并继续

### 第 2 步：执行批次
**默认：前 3 个任务**

对每个任务：
1. 标记为 in_progress
2. 精确遵循每个步骤（计划有细分步骤）
3. 按指定运行验证
4. 标记为 completed

### 第 3 步：报告
批次完成时：
- 展示实施了什么
- 展示验证输出
- 说："准备好接收反馈。"

### 第 4 步：继续
根据反馈：
- 如果需要应用更改
- 执行下一批次
- 重复直到完成

### 第 5 步：完成开发

所有任务完成并验证后：
- 宣布："我正在使用 finishing-a-development-branch 技能来完成这项工作。"
- **必需子技能：** 使用 superpowers:finishing-a-development-branch
- 遵循该技能来验证测试，呈现选项，执行选择

## 何时停下并寻求帮助

**立即停止执行当：**
- 批次中遇到阻碍（依赖缺失、测试失败、指令不清）
- 计划有关键缺口阻止开始
- 你不理解某个指令
- 验证反复失败

**寻求澄清而不是猜测。**

## 何时重访早期步骤

**返回审查（第 1 步）当：**
- 搭档根据你的反馈更新了计划
- 基本方法需要重新思考

**别强行通过阻碍** - 停下并询问。

## 记住
- 先批判性审查计划
- 精确遵循计划步骤
- 别跳过验证
- 计划说要引用技能时就引用
- 批次间：只报告然后等待
- 遇到阻碍时停下，别猜
